## 💡 SUCCESS TIPS

### Before You Start:
1. **Choose wisely:** Pick project that excites you but is achievable
2. **Plan first:** Break into small tasks (2-4 hours each)
3. **Start simple:** Get basic version working, then add features
4. **Commit often:** Every feature = new commit

### During Development:
1. **Test frequently:** Don't wait until end to test
2. **Use Git branches:** Make branch for each feature
3. **Write README as you go:** Document features as you build them
4. **Ask for help:** Use Discord, office hours, peers

### Before Submission:
1. **Test edge cases:** Try to break your program
2. **Read your code:** Does it make sense to fresh eyes?
3. **Check README:** Is everything explained clearly?
4. **Get peer review:** Fresh perspective catches bugs

---

## ⚠️ COMMON MISTAKES TO AVOID

❌ **Don't:**
- Wait until Sunday night to start
- Commit all code in one massive commit  
- Skip error handling ("it works on my machine")
- Ignore the README ("just look at my code")
- Copy code you don't understand
- Skip peer review process

✅ **Do:**
- Start Friday or Saturday
- Make small, frequent commits with clear messages
- Handle bad user input gracefully
- Write README for someone who's never seen your project
- Write every line yourself (or understand it fully)
- Take peer feedback seriously
